You need to display an SLA message

If the timeframe is more than 24 hours, you must display an SLA message.

For this, you need to raise an RFC request.

How to raise an RFC request

You can raise an RFC request by either:

  • posting a request in #cid-support Slack channel
  • or, submitting a Standard RFC form in #cid-release-prep Slack channel

You will need to provide the following details:

  • new timeframe
  • reason for change
  • content for the SLA message
  • when it needs to be done by

You can find step by step guidelines for updating SLA.

SLA messages

Content guidelines for entering date, time and SLA timeframe

Entering SLA timeframe

  • use the closest number of days rounded up when SLA is more than 24 hours i.e 2 days instead of 48 hours

Entering date

  • spell out months in full text i.e September
  • do not use '0' at the beginning of dates i.e 1 September 2022

Entering time

  • use 12 hr system i.e 1am / 1pm if it is on the hour OR 1.30am / 1.30pm if it is half past the hour
  • if it is 12pm, use 'midday'
  • if it is 12am, use 'midnight'
